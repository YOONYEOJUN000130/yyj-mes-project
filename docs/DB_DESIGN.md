# DB 설계 문서

## 1. 개념적 설계

본 프로젝트는 제조 현장에서 발생하는 설비 데이터를 가상 PLC가 생성하고, MES 서버가 이를 수집하여 Oracle DB에 저장한 뒤 생산 상태와 병목 공정을 분석하는 시스템이다.

관리 대상은 다음과 같다.

### 주요 개체

| 개체 | 설명 |
|---|---|
| 생산라인 | 제품이 생산되는 하나의 라인 |
| 공정 | 제품이 거치는 A/B/C 제조 단계 |
| 설비 | 각 공정에서 데이터를 발생시키는 장비 |
| 제품 | A/B/C 공정을 거쳐 완성되는 생산 대상 |
| PLC 이벤트 | 설비에서 발생한 공정 완료/불량/누락 관련 이벤트 |
| 생산 실적 | PLC 이벤트를 기반으로 저장되는 공정별 생산 기록 |
| 완성품 | A/B/C 공정을 모두 정상 통과한 제품 |
| 제품 진행 상태 | 제품별 A/B/C 공정의 OK/NG/미수신 상태 |

### 개체 간 관계

- 하나의 생산라인에는 여러 공정이 존재한다.
- 하나의 제품은 A/B/C 공정을 순서대로 거친다.
- 하나의 PLC 이벤트는 특정 제품, 특정 공정, 특정 설비에서 발생한다.
- MES 서버는 PLC 이벤트를 수신하여 생산 실적으로 저장한다.
- 제품이 A/B/C 공정을 모두 OK로 완료하면 완성품으로 등록된다.
- 공정 결과가 NG이거나 일부 공정 데이터가 수신되지 않으면 제품 진행 상태에서 DEFECT 또는 INCOMPLETE로 분류된다.
- 생산 실적의 사이클타임을 기반으로 공정별 병목을 분석한다.

### 개념적 데이터 흐름

```text
Virtual PLC
→ PLC 이벤트 생성
→ MES Server 수신
→ Oracle DB 저장
→ 생산 실적 / 완성품 / 제품 진행 상태 / 병목 분석
